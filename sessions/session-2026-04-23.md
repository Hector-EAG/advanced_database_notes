## What I understood
- When 2 transactions happen at the same time, the one that lock the record first is the one able to make changes to the record, and this is only lifted when the locked transaction makes a commit; this is an example of the consistency property of transactions. In regards of isolation as a property, it refersa as the changes made locally will remain local until a commit. For durability as a property, refers to the saving of the data on the disk. Finally atomicity as a property, refers to the concept of all or nothing, which meands that if one change before the commit is unavailable, none of the changes will commit. Can add save points as "replacements" for commits to use the roll back on, but not for updating the storage disk. Procedures help to standarizr porcesses to ensure that a commit is only done when all the data has been handle correctly. Finally functions are only working as a part of a SELECT, it will enrich the data retrieve from the SELECT.
