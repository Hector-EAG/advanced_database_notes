## Topics covered
- Indexes are data structures that improve the speed of data retrieval by reducing the need to do a full table scann.

## What I understood
- With time a full table scann becomes ineffective (slow), for this cases we use indexes. A type of index in the B-tree which saves in an index somo key numbers from which the search will based on (for example if you are searching for 60, and the index has the numbers 40 (0-40) and 80(41-80), it will ignore 40 given that its not in range and will start searching in the index 80). Another example is the hash index, which hashes each row to find it immidiatly, this makes it have a velocity of O(1), but it requires a lot of space given that for each row there is a hash code. There is also the composite index, which saves the rows with certain characteristics. There is also the partial index, as you crate it you will define what row characteristics will you save in the index. It is recomended to create an index at the foreign key.
